Apply UX review findings to ui-design.md and related design artifacts.

1. Read the latest ux-review.md in the Report Directory.
2. For each blocking finding, update ui-design.md (screens, states, navigation, interactions) to address the cited policy criterion.
3. Do not change implementation code — design documents only.
4. If a finding requires code changes, note it in the summary and leave implementation to review-fix-frontend.

When ui-design.md is absent, create or update the minimal design artifact needed to resolve UX findings.
