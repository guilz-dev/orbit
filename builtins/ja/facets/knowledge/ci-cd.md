# CI/CD知識

ビルド・テスト・デプロイのパイプラインパターン。`kubernetes-docker` と `observability` と併用する。

## パイプライン規律

| 基準 | 判定 |
|------|------|
| デプロイ前の自動テスト / 品質ゲートなし | REJECT |
| 再現不能ビルド（依存未固定、lockfile なし） | REJECT |
| パイプライン設定への秘密情報ハードコード | REJECT（secret store / OIDC を使う） |
| 決定的に再実行できないパイプライン | 警告 |

- すべてのデプロイを build + lint + test でゲートする。ビルドは再現可能に（依存固定、lockfile、action/image バージョン固定）。
- 秘密情報は secret store または OIDC 連携から注入する。パイプライン YAML にコミットしない。

## デプロイ安全性

| 基準 | 判定 |
|------|------|
| ロールバック経路のないデプロイ | REJECT |
| 本番前の staging / pre-prod 検証なし | 警告 |
| 手動の非スクリプト化デプロイ手順 | REJECT（自動化） |
| アーティファクト版管理 / トレーサビリティなし（commit → artifact → env） | 警告 |

- ロールバック（前アーティファクト / blue-green / canary）を用意する。各デプロイを commit と artifact に紐づける。
- 同一 artifact を環境間で昇格させる。環境ごとに再ビルドしない。

## アンチパターン

| アンチパターン | 問題 |
|--------------|---------|
| 開発者マシンからのデプロイ | 再現不能、監査不能 |
| 「 unblock のため」にテスト無効化 | リグレッションが出荷される |
| 環境ごとに別ビルド | 環境間ドリフト |
| キャッシュなしの長いモノリシックパイプライン | フィードバック遅い |
| ログへ echo される秘密情報 | 資格情報漏洩 |
