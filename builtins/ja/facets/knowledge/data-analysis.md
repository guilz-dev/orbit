# データ分析知識

データ分析は生データを意思決定に変換する活動である。意思決定に直結する問い、再現性、説明可能性を重視する。

## 分析ワークフロー

| 基準 | 判定 |
|------|------|
| 分析の問いや意思決定対象が明示されていない | REJECT |
| 手法を定義する前に結論を固定する | REJECT |
| 再実行不能な手作業スプレッドシートだけで分析する | 警告 |
| 単発スナップショットだけで傾向判断する | 警告 |

- 先に意思決定の問いを定義し、指標、母集団、期間、比較群を決める。
- 分析手順は SQL / notebook / script で再現可能にする。

## データ品質とバイアス確認

| 観点 | ガイダンス |
|------|------------|
| 欠損値 / null 処理 | 扱いルールを明示する |
| 重複レコード | 集計前に重複除去する |
| 時刻整合性 | timezone と event-time 境界を統一する |
| サンプリングバイアス | 一般化前に代表性を検証する |

## 伝達

| アンチパターン | 問題 |
|----------------|------|
| 平均値のみ提示して分布を示さない | 歪みや外れ値を隠す |
| 相関を因果として断定する | 意思決定根拠として不正確 |
| 不確実性の記述がない | 過剰確信につながる |

- 各結論に前提、制約、確信度を添えて報告する。
