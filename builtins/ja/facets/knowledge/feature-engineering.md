# 特徴量設計知識

特徴量設計はモデルが学習できる情報量を決める。特徴量数よりも、正しさと serving 一貫性を優先する。

## 設計原則

| 基準 | 判定 |
|------|------|
| 学習時と serving 時で特徴量定義が異なる | REJECT |
| 特徴量の意味や単位が曖昧 | 警告 |
| 正則化方針なしで高カーディナリティ特徴量を追加 | 警告 |
| 推論時に利用できない未来情報から特徴量を作る | REJECT |

- 特徴量ごとに source、変換、単位、鮮度を明記する。
- 脆い proxy を多数足すより、安定して説明可能な特徴量を優先する。

## 実務チェック

| 観点 | ガイダンス |
|------|------------|
| 欠損処理 | 特徴量ごとに default / imputation 方針を定義する |
| エンコード | 型とカーディナリティに応じて戦略を選ぶ |
| 時系列整合 | processing time ではなく event time 基準で特徴量化する |
| ドリフト感度 | 変動が大きい特徴量は監視強度を上げる |

## アンチパターン

| アンチパターン | 問題 |
|----------------|------|
| 同種の相関特徴量を無差別に追加 | 複雑性増加のみで信号増加が乏しい |
| notebook 内だけで重い前処理を実装 | 学習/推論の乖離が再現できない |
| 本番特徴量で feature store を迂回 | online/offline 値不一致 |
