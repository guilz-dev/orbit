# 機械学習知識

機械学習はモデル指標だけでなく、実際のプロダクト意思決定の改善を目的にする。

## 問題設定

| 基準 | 判定 |
|------|------|
| 予測対象、ラベル定義、活用アクションが不明確 | REJECT |
| ベースライン確認前に複雑モデルを選定する | 警告 |
| 学習目的と事業目的が一致していない | REJECT |
| 品質低下時のロールバック経路がない | 警告 |

- 予測を誰が使い、どの意思決定が変わり、失敗コストがどこまで許容されるかを先に定義する。
- ルールベースや単純統計などのベースラインを常に比較対象として持つ。

## データセットと分割戦略

| 観点 | ガイダンス |
|------|------------|
| ラベルリーク | 推論時に得られない未来情報や代理信号を除去する |
| train/validation/test 分割 | 実運用の時間順序やグループ境界に合わせる |
| クラス不均衡 | サンプリング/重み付けと評価指標を合わせて設計する |
| 特徴量可用性 | すべての特徴量が serving 時に取得可能であることを確認する |

## よくある失敗

| アンチパターン | 問題 |
|----------------|------|
| ベンチマーク指標だけを追う | 事業価値に結びつかない局所最適 |
| 同一 validation への過度な調整 | validation 過学習 |
| キャリブレーション無視 | 信頼度解釈が崩れる |
| デプロイ後監視を省略 | 品質劣化を見逃す |
