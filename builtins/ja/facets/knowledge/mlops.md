# MLOps知識

MLOps はデータ、モデル、基盤の変化に対して ML システムの信頼性を維持する実践である。

## デリバリと運用

| 基準 | 判定 |
|------|------|
| 学習成果物（コード、データ版、設定）が再現不能 | REJECT |
| 昇格基準なしの手動デプロイ | REJECT |
| データドリフト/予測品質の監視がない | REJECT |
| 前版モデルへのロールバック経路がない | 警告 |

- コード、データスナップショット、特徴量定義、モデル成果物、実行設定を一緒に版管理する。
- offline 検証、shadow、canary、全量展開の各段階に明示的ゲートを置く。

## 監視スコープ

| 信号 | ガイダンス |
|------|------------|
| Data drift | 重要特徴量の分布変化を追跡する |
| Concept drift | 入力と実結果の関係変化を監視する |
| Prediction quality | proxy 指標と遅延確定指標を併用する |
| System health | latency、throughput、error rate、resource 使用量を監視する |

## ガバナンスとリスク

| アンチパターン | 問題 |
|----------------|------|
| モデル版管理を任意扱いにする | 監査性とロールバック確信を失う |
| データから予測出力までの lineage を持たない | 原因調査が遅くなる |
| model/data pipeline のアクセス制御不足 | セキュリティ/コンプライアンス露出 |
| 品質劣化時の runbook 不在 | インシデント対応遅延 |
