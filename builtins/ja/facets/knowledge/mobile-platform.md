# モバイルプラットフォーム知識

UI フレームワークに依存しないモバイル横断課題（iOS & Android）。フレームワーク知識（`react-native` / `flutter` / `swiftui` / `jetpack-compose`）と併用する。

## アプリライフサイクルとバックグラウンド

| 基準 | 判定 |
|------|------|
| アプリが常にフォアグラウンドにある前提（バックグラウンド/復元未対応） | REJECT |
| メインスレッド上の長時間処理（jank / ANR） | REJECT |
| プロセス終了を跨いだ一時 UI 状態の永続化/復元がない | 警告 |
| バックグラウンド時にネットワーク/タイマーを止めない | 警告 |

- フォアグラウンド/バックグラウンド遷移を扱う。進行中状態を保存・復元する。
- メイン/UI スレッドをブロックしない。バックグラウンドスレッド/isolate へオフロードする。

## 権限

| 基準 | 判定 |
|------|------|
| 文脈に沿った説明なしで権限を要求 | REJECT |
| 拒否 / 「毎回確認」時の graceful な経路がない | REJECT |
| 必要になる前にまとめて権限を要求 | 警告 |
| 恒久的拒否状態の未対応（設定へのディープリンク） | 警告 |

- 権限は just-in-time に、理由を示して要求し、拒否時は劣化運用する。

## オフラインとデータ

| 基準 | 判定 |
|------|------|
| 接続前提でオフライン/失敗未対応 | REJECT |
| 不安定なネットワークでの失敗ミューテーションにリトライ/キューがない | 警告 |
| 平文ストレージ（Prefs/UserDefaults/AsyncStorage）への秘密情報/トークン | REJECT |
| 上限のないローカルキャッシュ増大 | 警告 |

- 断続的接続を前提に設計する: 読み取りをキャッシュ、書き込みをキュー、同期状態を表示。
- 資格情報は Keychain（iOS）/ Keystore（Android）に保存する。

## ナビゲーションと統合

| 関心事 | 指針 |
|---------|----------|
| ディープリンク / ユニバーサルリンク | エントリルートを定義・処理し、パラメータを検証 |
| プッシュ通知 | フォアグラウンド/バックグラウンド配信とタップルーティングを処理 |
| 戻る操作（Android） | システム戻るを尊重。ユーザーを閉じ込めない |
| バッテリー / 位置情報 | 細かい精度が不要なら粗い精度を使い、バックグラウンドでは停止 |

## アンチパターン

| アンチパターン | 問題 |
|--------------|---------|
| セーフエリア / ノッチ / ジェスチャーバー無視 | UI の欠け・到達不能 |
| I/O で UI スレッドをブロック | jank、ANR、watchdog 終了 |
| 状態復元なし | OS がプロセスを回収した後に作業喪失 |
| 権限の過剰要求 | ストア拒否、ユーザー不信 |
