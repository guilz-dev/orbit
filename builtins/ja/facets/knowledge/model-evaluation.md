# モデル評価知識

モデル評価は実際の意思決定文脈と一致している必要がある。指標が良くても意思決定品質が悪ければ不十分。

## 指標と目的の整合

| 基準 | 判定 |
|------|------|
| 指標が事業目的に対応していない | REJECT |
| トレードオフがあるのに単一指標のみで判断 | 警告 |
| 閾値選定理由がない | 警告 |
| 重要セグメント別評価がない | REJECT |

- 主要指標、ガードレール指標、運用閾値をセットで定義する。
- 地域、デバイス、ユーザー種別、リスク層など影響差が大きい軸で分解評価する。

## 検証戦略

| 観点 | ガイダンス |
|------|------------|
| holdout 設計 | 時系列とグループ漏洩境界を維持する |
| 交差検証 | データ量や分散が大きい場合に頑健性確認として使う |
| 統計的信頼性 | 点推定だけでなく区間で報告する |
| ベースライン比較 | baseline と現行本番モデル差分を明示する |

## 失敗分析

| アンチパターン | 問題 |
|----------------|------|
| offline 指標だけでリリース判断 | offline/online ギャップを無視 |
| 偽陽性/偽陰性の分析なし | 誤判定コストが不明 |
| キャリブレーションやランキング品質を無視 | downstream 判断品質低下 |
| shadow/canary なしで本番投入 | リリースリスク増大 |
