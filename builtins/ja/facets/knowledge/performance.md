# パフォーマンス知識

横断的なパフォーマンスのパターン。記述的。最適化の前に計測する。

## 手法

| 基準 | 判定 |
|------|------|
| 計測/プロファイルなしの最適化 | REJECT（早すぎる） |
| パフォーマンス変更の前後にベースラインメトリクスがない | 警告 |
| ホットパスを無視してコールドパスをマイクロ最適化 | 警告 |
| 無効化戦略なしのキャッシュ追加 | REJECT |

- まず計測する（プロファイラ、トレース、クエリログ）。証明されたホットパスを最適化し、再計測する。
- すべてのキャッシュには定義された TTL と無効化ルールが必要。古いデータは正しさのバグである。

## よくあるホットスポット

| 領域 | パターン |
|------|---------|
| データアクセス | N+1 クエリ → バッチ/join。欠落インデックス → 追加。`SELECT *` → 列を絞る |
| I/O | 並列化できる逐次 await。チャッティーなネットワーク往復 |
| CPU | メモ化できる反復作業。ループ内で持ち上げられる処理 |
| ペイロード | 過剰取得。ページングなし。非圧縮レスポンス |
| メモリ | 上限のないキャッシュ/コレクション。解除漏れリスナーによるリーク |

| 基準 | 判定 |
|------|------|
| ホットパス上の N+1 クエリパターン | REJECT |
| レイテンシクリティカルな非同期パス上のブロッキング呼び出し | REJECT |
| 集計のためにデータセット全体をロード | REJECT（ストアへ押し下げる） |
| 上限のないインメモリキャッシュ（eviction なし） | REJECT |

## アンチパターン

| アンチパターン | 問題 |
|--------------|---------|
| 早すぎる最適化 | 計測利益なしの複雑化 |
| 無効化なしのキャッシュ | 古い/誤ったデータ |
| ハードウェア追加だけで perf を直す | 真のコストを隠す |
| 未証明の速度のために可読性を犠牲にする | 保守性低下 |
