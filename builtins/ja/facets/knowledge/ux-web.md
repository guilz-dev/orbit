# UX知識 — Web

Web 固有の UX 規約。汎用 `ux` 知識と `frontend` と併用する。

## レスポンシブと入力

| 基準 | 判定 |
|------|------|
| 一般的なブレークポイントでレイアウト破綻 / 非レスポンシブ | REJECT |
| マウス専用インタラクション（キーボード相当なし） | REJECT |
| タッチ/フォーカス相当なしの hover 専用 affordance | REJECT |
| ズーム / テキスト拡大を無視した固定ピクセルレイアウト | 警告 |

- ビューポートサイズに対応するレスポンシブレイアウトを設計し、ポインタとキーボードの両方をサポートする。
- すべての hover affordance には focus/touch 相当が必要。

## ナビゲーションとフォーム

| 基準 | 判定 |
|------|------|
| ブラウザ戻る/履歴の破壊（適切なルーティングなし SPA） | REJECT |
| インラインバリデーションと明確なエラーメッセージのないフォーム | REJECT |
| ローディング/無効化状態なしの送信（二重送信） | REJECT |
| 警告なしのナビゲーション/更新での入力データ喪失 | 警告 |

- ブラウザの戻る/進むとディープリンク可能な URL を尊重する。
- フォーム: 全フィールドにラベル、インライン検証、フィールド近くのエラー表示、二重送信防止。

## アクセシビリティとパフォーマンス（体感）

| 基準 | 判定 |
|------|------|
| キーボード操作不能 / フォーカス順の破綻 | REJECT |
| WCAG コントラスト不合格 | REJECT |
| ルート変更 / モーダルでフォーカス管理なし | REJECT |
| 遅いロードに skeleton/楽観的フィードバックなし | 警告 |
| コンテンツロード時のレイアウトシフト（CLS） | 警告 |

- WCAG AA を満たす: キーボード操作、フォーカス管理、コントラスト、セマンティックランドマーク。
- skeleton/楽観的 UI で体感レイテンシを下げ、レイアウトシフトを避ける。
