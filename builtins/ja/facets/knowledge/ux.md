# UX知識

UX のパターンとアンチパターン（記述的）。強制は UX ポリシー側。この facet は判断を補う。

## 画面状態

データ連動または非同期の画面には、ハッピーパス以外の状態がある。すべて設計する。

| 状態 | いつ | 必ず示すもの |
|-------|------|-----------|
| Loading | データ取得中 | 画面全体ではなく影響領域にスコープしたスケルトンまたはスピナー |
| Empty | クエリが何も返さない | 原因 + 次のアクション（「結果がありません。フィルタを解除してください。」） |
| Error | リクエストまたは操作が失敗 | 何が失敗したか + リトライ経路。空白画面にしない |
| No-permission | ユーザーに権限がない | なぜ非表示か + 誰に聞けばよいか。死んだ 403 にしない |
| Partial | 一部ロード、一部失敗 | ロード済みを表示し、失敗領域を明示 |

| 基準 | 判定 |
|------|------|
| ハッピーパスだけの非同期画面 | REJECT |
| 既にロード済みコンテンツを遮る全画面スピナー | 警告 |
| 次のアクションのない空状態 | 警告 |

## ナビゲーションと到達可能性

| 基準 | 判定 |
|------|------|
| 画面は存在するが、メニュー/ルート/アクションから到達できない | REJECT |
| 完了、終了、戻り経路のないフロー | REJECT |
| 警告なしに戻る/キャンセルでユーザー入力を破棄 | 警告 |
| 主要アクションが視覚的に主要でない | 警告 |

## インタラクションとフィードバック

- すべてのユーザー操作に即時フィードバック（状態変化、トースト、インライン結果）。
- 破壊的操作は確認または undo 可能にする。
- 無効化されたコントロールは理由を説明する（ツールチップ/ヘルパーテキスト）。黙って無効にしない。
- ボタンのローディングはそのボタンに局所化し、ページ全体にしない。

## 情報設計

- 画面ごとに 1 つの主要アクション。他は格下げする。
- 段階的開示: 高度なオプションは必要になるまで隠す。
- 関連フィールドをグループ化し、DB スキーマではなくユーザーのメンタルモデルに合わせる。
- ユーザーが判断に必要なものだけを示し、残りは後回しにする。

## アクセシビリティ（設計時）

| 基準 | 判定 |
|------|------|
| アクセシブル名のないインタラクティブ要素 | REJECT |
| 色だけで意味を伝える | REJECT |
| モーダル開閉でフォーカスを閉じ込め/復元しない | REJECT |
| タッチで ~44px 未満のタップ/クリック領域 | 警告 |

## アンチパターン

| アンチパターン | 問題 |
|--------------|---------|
| Mystery meat navigation | ラベルなしアイコン。ユーザーが推測 |
| Dead-end error | リトライや戻り経路のない失敗 |
| Silent failure | エラー時に目に見える変化がない |
| Modal stacking | ダイアログの上にダイアログ。明確な脱出なし |
| Schema-shaped form | テーブル列順のフィールド。タスク順ではない |
| Infinite spinner | タイムアウトやエラー fallback のないローディング |
